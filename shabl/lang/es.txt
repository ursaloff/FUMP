[T_SENDING_TIPS]	
	<h2>Configurando encabezado de rebotes. </h2>
                    <p>Si va a procesar mensajes rebotados puede definir 
                    las opciones de envío de rebotes allí.</p>
                    <p>&nbsp;</p>
                    <p><font color="#FF0000"><strong>Atención!</strong></font> 
                      Los mensajes rebotados pueden ser regresados con la configuración
                      Pre-determinada, necesitara definir las opciones de rebote si no encuentra
                      el mensaje rebotado en su buzón o necesita especificar el buzón que debe
                      ser diferente al sus encabezados Para o Responder a. Buena suerte!</p>
                    <p>Usualmente todos los mensajes rebotados regresan al correo en el Return-path
                     en su mensaje.  Hay algunas maneras de implementar esto, algunas se prefieren
                     sobre otras.</p>
                    <p><strong><u>SMTP</u></strong></p>
                    <p>Si esta utilizando SMTP para enviar mensajes puede especificar
                    un correo diferente al de los encabezados "De" o "Responder a" 
                    Necesita marcar 'Utilice el correo y SMTP' y especificar el correo.<br>
                      <br>
                      <strong><u>Configuración usando el comando SendMail</u></strong></p>
                    <p>Sus resultados serán diferentes con este método pero vale la pena probarlo: </p>
                    <p><br>
                      <strong><em>Técnica 1:</em></strong> The -f flag</p>
                    <p>Marcar 'Use sendmail -f flag' y especifica un correo allí</p>
                    <p>Esto debería configurar el encabezado Return-Path</p>
                    <p>&nbsp;</p>
                    <p><em><strong>Técnica 2: </strong></em>Encabezado Return-Path </p>
                    <p>If that doesn't work, you can try to set the Return-Path 
                      header explicitly. This is not the best way by any means, 
                      but Qmail allows you to do this.</p>
                    <p>&nbsp;</p>
                    <p><strong> Técnica  3: </strong>Errors-To header</p>
                    <p>El Errors-To header parece haber sido creado solamente
                    para este caso, pero parece ser ya un encabezado rechazado, úselo con 
                    precaución. Para que funcione, deberá configurar
                      Sendmail para que lea los Errors-To header. No recomiendo
                      usar estos encabezados, pero si cualquiera de las técnicas anteriores falla
                      puede probar el Errors header.</p>
                    <p>&nbsp;</p>
 <p>Para probar cualquiera de estas configuraciones, Envíese un correo Usted mismo y vea el source (datos de origen= del mensaje, 
en su programa de correos (ej. Outlook). En los encabezados, Deberá ver el Return-Path header parecido al siguiente:</p>
                    <blockquote>
                      <p><code>Return-Path: &lt;info-store@sellwide.net&gt;<br>
                        Delivered-To: helpdesk@sellwide.net<br>
                        Content-Disposition: inline<br>
                        Content-Transfer-Encoding: binary<br>
                        Content-Type: text/plain<br>
                        MIME-Version: 1.0<br>
                        X-Mailer: MIME::Lite 2.117 (F2.71)<br>
                        Date: Tue, 14 Jun 2005 15:26:41 UT<br>
                        From: FUMP Registration &lt;info-store@sellwide.net&gt;<br>
                        To: Konstantin Ursaloff &lt;info-store@sellwide.net&gt;<br>
                        Subject: User mom_id2000@yahoo.com registered at the dlfumlpl 
                        account </code></p>
                    </blockquote>

<p>Aquí puede editar las cuentas POP3 para coleccionar mensajes rebotados
                    y definir las reglas/acciones de prospectos (remover o desactivar
                    prospectos):<br>
                    <br>
                    <strong>Servidor POP3</strong> - su servidor POP3 para coleccionar 
                    mensajes</p>
                  <p><strong>Puerto POP3</strong> - puerto predeterminado es el 110</p>
                  <p><strong>Usuario POP3</strong> - usuario para ingresar al buzón</p>
                  <p><strong>Contraseña POP3</strong> - contraseña para ingresar al buzon<br>
                  </p>
                  <p><strong>Remover&nbsp;prospectos&nbsp;despues</strong>:<br>
                    - <strong>rebotes duros:</strong> Este es el conteo de mensajes con
                    rebotes duros antes que los prospectos sean removidos o desactivados
                    (Rebotes duros son errores permanentes como servidor o usuario no encontrado
                    rechazo a spam, y significa que su mensaje nunca será recibido por el 
                    recipiente). FUMP esta pre-configurado para remover/desactivar prospectos despues
                    de un rebote duro pero puede cambiar esta regla</p>
                  <p>- <strong>rebotes suaves:</strong> conteo de rebotes suaves antes que prospectos
                  sean removidos o desactivados (Rebotes suaves son errores temporales como cuando el
                  buzón esta lleno, etc.). FUMP esta pre-configurado para remover/desactivar prospectos después
                    de tres rebotes suaves pero puede cambiar esta regla.</p>
                  <p><strong><br>
                    Acciones: </strong></p>
                  <p>- <strong>remover prospectos de la base de datos:</strong> sus prospectos serán
                  removidos de la base de datos</p>
                  <p>- <strong>cambiar estatus a desactivado:</strong> sus prospectos serán
                  desactivados</p>
                  <p>- <strong>Agregar prospecto a la lista negra:</strong> 
                    Puede configurar esta opción para automáticamente agregar prospectos a la lista
                    negra cuando sean removidos o desactivados</p>
